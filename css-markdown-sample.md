### 1.1.1. CSSクラス付きのセクション

Markdown内でHTML要素とクラスを併用すると、デザインを細かく制御できます。

- `section-card` でボックスに背景と角丸を追加
- `highlight` で強調ラベルを作成
- `button-primary` でリンクをボタン風に表示

例:

## 1.3. Markdown記法の詳細

Markdownの基本構文を詳しく理解すると、ドキュメントを自由に書けるようになります。以下は、見出し、段落、リスト、引用、コード、表、リンク、画像などの詳細な例です。

### 1.3.1. 見出し（H1～H6）

# 2. 見出し H1

## 2.1. 見出し H2

### 2.1.1. 見出し H3

#### 2.1.1.1. 見出し H4

##### 2.1.1.1.1. 見出し H5

###### 2.1.1.1.1.1. 見出し H6

H1 / H2 は `=` または `-` を使うアンダーライン記法でも書けます。

例:

1. 見出し H1
=========

3.1. 見出し H2
---------

### 3.1.1. 長い文章の例

Markdownでは長い文章をそのまま書いても、段落として正しくレンダリングされます。複数行の途中で改行したい場合は、行末に半角スペースを2つ入れます。

この文章は長い段落の例です。Markdownは改行コードだけでは段落を分割しないため、文章を読みやすく書きたいときは自然な位置で改行してください。行の終わりに2つの半角スペースを入れると、ブラウザ上で折り返し改行が反映されます。

このように、Markdownの段落は連続したテキストとして扱われるため、長文でもきれいに書けます。

### 3.1.2. リストの例

- 箇条書き 1
- 箇条書き 2
  - ネストした箇条書き 2-1
  - ネストした箇条書き 2-2
- 箇条書き 3

- アスタリスクでも箇条書きを書けます

- プラス記号も同じです

1. 番号付きリスト
2. 2 行目
   1. インデントされた番号付きリスト
   2. さらにサブ項目
3. 3 行目

タスクリストも書けます。対応するレンダラーではチェックボックスになります。

- [x] 完了した項目
- [ ] 未完了の項目

## 3.4. 実践例

1. `<style>` ブロックをMarkdown内に埋め込む
2. HTMLの`div`や`span`を使ってクラスを割り当てる
3. Markdownの見出しやリストと組み合わせて、自然な文書として表示する

### 3.4.1. 使い方のポイント

- GitHubのREADMEでは`<style>`は無効化される場合があります
- 静的サイトジェネレーターやMarkdown対応CMSなら、ほとんどの場合で有効です
- 外部CSSファイルを使う場合は、レンダリング環境が`<link>`タグを許可している必要があります

## 3.5. Markdownの右寄せ表記法

Markdownの標準テーブル記法では、列のセル配置を指定できます。右寄せにしたい列では、区切り線のコロンを右側に付けます。

例: 1 列目は左寄せ、2 列目は右寄せ

| 項目 | 数値 |
| :--- | ---: |
| 売上 | 1,234 |
| 利益 | 567 |
| 成長率 | 12.3% |

- `:---` は左寄せ
- `:---:` は中央寄せ
- `---:` は右寄せ

テーブル全体を右側に寄せるMarkdown記法は標準では存在しません。代わりに、右寄せしたいセルや列を指定するのが一般的です。

### 3.5.1. 表を右寄せ風に見せる補助テク

Markdown自体で「表全体を画面右側に寄せる」記法はありませんが、HTMLを使える環境では次のように `align="right"` や CSS を使って配置できます。

```html
<div style="text-align: right;">
  <table>
    <tr><th>項目</th><th>数値</th></tr>
    <tr><td>売上</td><td>1,234</td></tr>
  </table>
</div>
```

しかし、純粋なMarkdownで使うなら、表の中身を右寄せする `---:` を覚えておくとよいでしょう。

## 3.6. 印刷を考慮したMarkdownとCSS

印刷を考える場合、以下の点を意識すると見栄えがよくなります。

- 背景色や影は印刷で消えることがあるため、白背景と黒文字を基本にする
- リンク先は印刷時に残すため、`a::after` でURLを表示する
- 表やコードブロックはページ分割に強い書き方をする
- `@media print` で印刷専用スタイルを用意する
- `@page` で余白を設定し、ページレイアウトを安定させる

このサンプルの `assets/markdown-pdf.css` には、印刷時に背景を透明化し、リンクURLを追記し、ページ分割を改善するスタイルが含まれています。
